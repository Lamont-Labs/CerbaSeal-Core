# Non-Goals

This repository intentionally does NOT implement the following:

## 1. Identity and Authentication Systems
Handled by upstream systems. CerbaSeal consumes verified identity context.

## 2. Policy Authoring Engine
Policy semantics are not defined here. Only enforcement structure is demonstrated.

## 3. Infrastructure / Hosting Model
Deployment topology is defined during pilot scoping.

## 4. Data Persistence Layer
Audit log is in-memory for demonstration. Persistence is environment-specific.

## 5. UI or Dashboard
CerbaSeal is not a UI-driven system.

## 6. Full Compliance Certification
This repository does not claim regulatory certification.

## 7. Client-Specific Implementation
All real workflows, integrations, and policies are defined during paid pilot scope.

---

## Why this matters

CerbaSeal is presented as a bounded enforcement layer, not a full product.
